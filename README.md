  `Farma System — CRM для складского учета и логистики аптечной сети`
Пет-проект по автоматизации складских остатков, закупок у поставщиков и розничных заказов.
Система построена на базе NestJS и React, реализует сложную бизнес-логику связей между модулями.
___
🚀 `Основной стек`
Backend: NestJS, TypeScript, Prisma ORM, PostgreSQL.
Frontend: React, TypeScript, Tailwind CSS, Axios.
Архитектура: REST API, транзакционная логика (ACID), модульная структура.
___
`🛠 Реализованный функционал`
1. `Управление складом (Inventory Management)`
  - Отображение актуальных остатков лекарственных средств.
  - Динамическая история: Автоматическая фиксация каждого движения товара (поступление/списание).
  - Система уведомлений о критическом остатке (визуальная индикация).
2. `Модуль закупок (Supply Chain)`
  - Формирование заказов поставщикам.
  - Транзакционная приемка: При создании заказа на поставку данные на складе обновляются атомарно через $transaction в Prisma.
  - Автоматическое формирование логов INCOMING для истории склада.
3. `Модуль заказов (Sales)`
  - Интерфейс «Корзины» для оформления продаж клиентам.
  - Validation Logic: Проверка наличия товара на складе перед списанием на уровне БД.
  - Автоматический расчет итоговой стоимости и обновление остатков.
4. `Безопасность и роли`
  - Разграничение прав доступа (Admin / User).
  - Шифрование паролей и регистрация пользователей.
___
`🏗 Технические особенности`
1. `Транзакции в БД:` Использование транзакций для обеспечения целостности данных (нельзя создать заказ, если не удалось обновить склад).
2. `DTO & Validation:` Строгая типизация входящих данных на бэкенде.
3. `Relational DB Schema:` Спроектирована база данных с отношениями "Один-ко-многим" (Товар -> История, Заказ -> Позиции).
4. `Shared Types:` Использование общих интерфейсов TypeScript между фронтендом и бэкендом для минимизации ошибок.
5. `Custom UI Components:` Интерфейс построен на Tailwind CSS с использованием Side Drawers для детальной аналитики товара.

`⚙️ Как это работает (Кратко)`
Проект демонстрирует навыки проектирования Fullstack-приложений с упором на надежность данных.
В коде реализована логика, исключающая появление "товаров-призраков" или некорректных списаний за счет использования возможностей Prisma ORM и NestJS Providers.
